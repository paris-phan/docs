Git and collaboration conventions are documented in a synced `AGENTS.md` carried identically across every repo in a venture. Linear is the spine.

## Linear is the unit of work

Issue tracking drives the workflow (team **Engineering**, initiative **GreekCore v2**). Every git artifact links back to an issue.

<Steps>
  <Step title="Start from a Linear issue">
    Work begins as a Linear issue, not a branch.
  </Step>
  <Step title="Copy the generated branch name verbatim">
    Branches use Linear's generated name exactly — e.g. `paris/eng-175-ios-app-store-submission-release`.
  </Step>
  <Step title="Commit in Conventional Commits">
    `feat:` / `fix:` / `chore:` …, no emoji, written so they "read like a person dashed it off," with `[part of ENG-XX]` appended.
  </Step>
  <Step title="Open a walkthrough-style PR">
    The PR body walks the reviewer through the change and includes `Part of ENG-XX` for auto-linking; reviews are right-sized.
  </Step>
</Steps>

## Commit style

<CardGroup cols={2}>
  <Card title="Conventional Commits" icon="git-commit-vertical">
    Typed prefixes (`feat`, `fix`, `chore`, …) with no emoji.
  </Card>
  <Card title="Human voice" icon="message-square">
    Messages read like a person dashed them off — not machine-generated boilerplate.
  </Card>
  <Card title="Issue linkage" icon="link">
    Append `[part of ENG-XX]` so commits trace back to Linear.
  </Card>
  <Card title="Right-sized PRs" icon="git-pull-request">
    `Part of ENG-XX` in the body for auto-linking; review scope matched to the change.
  </Card>
</CardGroup>

## The `CLAUDE.md` → `AGENTS.md` pattern

A one-line `CLAUDE.md` imports the real `AGENTS.md`, which is the single source of truth — and agent-agnostic, so it serves Claude Code, Cursor, and any other agent.

```md CLAUDE.md
@AGENTS.md
```

<Note>
  Writing the rules once in `AGENTS.md` means every agent reads the same instructions, and the file can be carried identically across all repos in a venture.
</Note>
